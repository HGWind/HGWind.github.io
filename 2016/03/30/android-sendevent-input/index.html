      • Android代码模拟屏幕点击

      Post on 2016-03-30

      本文讲的是通过使用代码，在整个android设备模拟屏幕点击。

      getevent/sendevent/input

      android系统下的一个工具，可以模拟多种按键和触屏操作。getevent是获取，sendevent/input是发送。具体不深究，可以自己查资料。可参考Android通过代码模拟物理、屏幕点击事件

      使用adb和设备连接，命令adb shell

      getevent(用来分析事件发送的格式)

      • 系统接收到的event，在/system/bin/下面，在adb shell下运行getevent查看。(注：显示的数据是16进制)
      • 调用getevent -l做同样的操作,返回的信息会以常量的名字出现。

      设备的不同，事件的格式不一样。需要自己分析。

      事件格式分析：

      之后补充，可参考Android跨进程模拟触屏事件(sendevent)
      ，写的挺好的，但是只针对单一设备分析，且没有考虑旋转屏情况。

      sendevent

      根据getevent的数据，将其数值翻译成10进制，以同样的格式发送，就可以产生同样的事件。

      input

      • 可以做到点击、滑动、按键的效果。

      比sendevent使用简单，且屏幕反向改变后坐标自己会调整；但是没有sendevent使用灵活（长按，多点触控难以做到），且反应迟钝。

      小结

      • input使用有局限性，使用简单暴力，没什么好说的；
      • getevent 和 getevent -l 用来分析event格式；
      • sendevent 用来产生event。

      点击屏幕的位置坐标

      通过命adb shell getevent -p获得event 体系里 宽（0035）和高（0036）
      0035 : value 0, min 0, max 32768,
      0036 : value 0, min 0, max 32768,

      • 网上找的的计算公式：
        据说是由androidvncserver源码，反推而来。
        x = (x - min) screenWidth / (max - min);
        y = (y - min)         screenHeight / (max - min);

      但是公式并没有考虑到屏幕旋转的情况。
      屏幕旋转后，坐标会改变。所以需要获取屏幕旋转情况。

      获取屏幕旋转

      int rotation = ((WindowManager)getSystemService(Context.WINDOW_SERVICE)).getDefaultDisplay().getRotation();
      测试结论：顺时针旋转，依次0，1，2，3

      屏幕点击坐标计算

      直接给结论，解析放后面，若发现不能正确使用，自行分析。

      screenWidth = w， screenHeight = h，min = 0， max = m;
      (x_0, y_0)要触摸的点坐标

      rotation : 0
      x = mx_0 / w; y = my_0 / h
      rotation : 1
      y = m(w - y_0) / w ; x = mx_0 / h
      rotation : 3
      x = my_0 / w; y = m(h - x_0) / h
